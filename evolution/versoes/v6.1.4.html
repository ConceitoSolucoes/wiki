S=24219 Foi identificado que no Pedidos/Orçamentos o limite máximo é de 13 (treze) caracteres para o código identificador, porem o caixa só permite filtrar por até 10 (dez) caracteres. Feito o ajuste.

 

 

 S=24228 Ao informar uma administradora de cartão inativa, o sistema estava apresentando os dados indevidamente. Feito o ajuste para que o sistema alerte que a administradora está inativa e/ou inexistente, e também para que o cursor volte para o controle onde se define a administradora.

 

S=24230 Durante a abertura do formulário de sugestões do pedido de compra (F10), após o carregamento de dados, se o usuário clicasse na parte externa do formulário principal, ele era apresentado novamente, colocando o formulário de sugestão atrás da aplicação, o que impedia que fosse aberto novamente, ocorrendo duplicidade e ocasionando outras falhas no avanço das etapas. Após a identificação, foi feito o ajuste.

 

S=24231 Ao definir um item no pedido de compra, se configurado para tal, o sistema apresenta um informativo de estoque, porém quando item estiver sendo incluso através da confirmação por sugestão de compra (F10), o sistema apresentava essas confirmações, porém elas não são necessárias ( pois já estão listadas como sugestão de compra), o que causava lentidão no sistema. Ajustado para cada item não apresentar informativo de estoque quando inclusos através de sugestão de compras (F10) , tornando essa etapa mais rápida.
